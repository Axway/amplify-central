## Consumer experience

### Marketplace

The following enhancements and bug fixes are available in this release.

#### Feature updates for Marketplace

* **Filter pagination** - The filter on Marketplace have been enhanced to manage the pagination. Now, you will be able to scroll 3 times on a filters to see the current values. Then a **Load more** button appears to load the next set of data. Also, an indicator of the number of data for that filter is displayed on the top right corner of it. Refer to [Search and filter Marketplace content](/docs/manage_marketplace/consumer_experience/searching#filter-a-result)
* **REason of Subscription cancellation** - The reason of subscription cancellation is now displayed in the subscription details so that consumer are aware why their subscription have been terminated. This cancellation can happen in 2 circumstances: when the subscription reached its term or when provider has activated the automatic subscription termination upon past due invoices.

#### Bug fixes for Marketplace

| Case ID   | Internal ID  | Description                                       |
|-----------|--------------|---------------------------------------------------|
| -         | APIGOV-29004 | **Issue**: Marketplace with vanity url cannot change the Marketplace language. <br />**Resolution**: A fix related to the cross domain issue has been added to correctly manage the language update from the Marketplace language selector. |
| -         | APIGOV-29002 | **Issue**: Resource names not visible in the credentials side blade for consumer org users. <br />**Resolution**: An API parameter was missing and prevented to view the resources associated to a credential. The parameter is now part of the request and enable the consumer to see the corresponding resources. |
| 01642315  | APIGOV-28923 | **Issue**: API Marketplace API - wrong ratingInfos. <br />**Resolution**: The GET product API has been fixed to ensure a consistent return information regarding the product ratings data. |
| -         | APIGOV-28809 | **Issue**: Errors in multiple UI screens in marketplace if not the latest product version is selected. <br />**Resolution**: The API to display the data has been fixed as well as an information message telling the user he is not visualizing the latest release when navigating to Resources/Documentation/plans screens. |
