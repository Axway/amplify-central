### On-premise agents

The helm based deployment of current and future On-prmise agents have been updated to use agent images from the [https://docker.repository.axway.com](https://docker.repository.axway.com). The use of a kubernetes secret is required.  This change has been made to the Axway API Management Gateway Agent, Istio, Graylog and Tracable on-premise agents.
For more details, see the:
[Connect API Manager](https://content/en/docs/connect_manage_environ/connect_api_manager/deploy-your-agents-with-amplify-cli.md)
[Connect Nesh Management](https://content/en/docs/connect_manage_environ/mesh_management/deploy-agents-with-helm.md)
[Connect Graylog](https://content/en/docs/content/en/docs/runtime_security/configure_runtime_compliance_graylog.md)
[Connect Traceable](https://content/en/docs/content/en/docs/runtime_security/configure_runtime_compliance_traceable.md)

##### Enhancements for Azure Agent

There are no enhancements in this release.

##### Bug fixes for Azure Agent

| Case ID     | Internal ID  | Description                                       |
|-------------|--------------|---------------------------------------------------|
| -           | APIGOV-28994 | **Issue**: The Azure Discovery and Traceability Agents Version 1.3.1 would fail to start up if the authentication key configuration is not set (CENTRAL_AUTH_PRIVATEKEY & CENTRAL_AUTH_PUBLICKEY). <br />**Resolution**: A fix was made to allow the Azure Agents to startup without the authentication key configuration. |
| -           | APIGOV-29038 | **Issue**: The Azure Discovery Agent would discover all APIs without honoring the filter or tag settings. <br />**Resolution**: A fix was made to the Azure Discovery Agent to honor the configured filter and tag settings. |
