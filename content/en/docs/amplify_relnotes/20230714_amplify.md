### Product Foundry

The following enhancements and bug fixes are available in this release.

#### Feature updates for Product Foundry

* None.

#### Bug fixes for Product Foundry

* **Product plan creation does not stay in the Product Plan section** - When a plan is created, the user is now redirected to the Product Plan section to see the result of its creation.
* **Impossible to create a plan** - after the user saved his plan, a refresh has been added to enable fetching the correct plan status. Since the treatment of the plan creation could be long, the refresh action is automatically performed 3 times with a delay of 2 seconds between each tentative.
* **Wrong Marketplace publication status** - With the multi-marketplace, the product publication status was not accurate. Now, the product publication status will be PUBLISHED as soon as the product is published in one or more Marketplace. Otherwise the status will remain UNPUBLISHED.
* **The publication of a product fails when including plan in invalid state** - The publication screen has been enriched to prevent the user from activating an invalid plan during the product publication process. More, in case, there are no more plan to activate when publishing a product, the user will see a message with the number of plan already activated.
* **Category deletion popup enrichment** - The delete category confirmation popup is now displaying the list of category that are planned to be deleted.

#### Known issues for Product Foundry

There are no known limitations in this release.

## Consumer experience

### Marketplace

The following enhancements and bug fixes are available in this release.

#### Feature updates for Marketplace

* **Display the actual swagger version of the Service** - The resource version displayed is coming from the the underline service swagger (`info.version`).

#### Bug fixes for Marketplace

* **Switching from a public Marketplace to a private one shows error** - Each time a Marketplace is open, the current user session is checked to ensure the user is able to access the Marketplace. In case the Marketplace he wants to access does not match his current session, a new page is displayed to allow him either the switch organization or logout.
